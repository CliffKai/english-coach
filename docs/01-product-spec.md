# 产品需求规格

## 功能 1：生词收集

**流程**：用户输入英文文本 → TokenizerAgent 切词 → 词形还原（lemma）→ 去重 → 按词频/用户水平基线过滤掉过于简单的词 → 逐个问用户「认识 / 跳过 / 不认识」。

- "跳过" / "认识"：不处理。
- "不认识"：连同**来源句**一起存入生词本（`VocabEntry`），供功能3背诵。

**补充设计**：
- 切词太碎的优化：先按词频剔除 the/is 等高频词，再按水平基线只问"可能不认识"的；支持"本段都认识"批量跳过。
- 多义词/重复词：同一个词在不同文章出现且义项不同，存**多条来源句**（同 word 多 context）。

## 功能 2：话题练习

用户给定话题，选择四种模式之一：

| 模式 | 代号 | 纠错策略 |
|---|---|---|
| a. 引导式写作 | `guided_write` | **练习模式**：即时纠错 + 脚手架引导 |
| b. 引导式口语 | `guided_speak` | **练习模式**：即时纠错 + 脚手架引导 |
| c. 自由写作打分 | `free_write` | **考试模式**：延迟纠错 |
| d. 对话打分 | `dialogue` | **考试模式**：延迟纠错 |

**延迟纠错状态机**（考试模式核心）：

```
考试模式：
  用户输出 → 静默记录错误到 buffer（不打断）→ 继续
  ... 直到用户点"完成"或"提前交卷" ...
  → 一次性输出：① 雅思/托福各维度分数
               ② 错误清单（原句 + 修正 + 解释）
               ③ 错误模式总结
               ④ errors 写入错题本
```

**规则**：
- 考试模式**零脚手架**，不救场、不给提示——压力是刻意设计，避免依赖。
- 但允许**随时"提前交卷"**：按已有内容打分+复盘（考试本就能提前交卷，这不是救场）。
- 卡顿/停顿本身是流利度评分信号，应记录进复盘，不抹平。

## 功能 3：理解式背单词

| 子功能 | 代号 | 逻辑 |
|---|---|---|
| a. 逐词理解背 | `recall_explain` | agent 给出来源句，问"这个词在这句里你理解是什么意思"，用户用自己的话说，agent 判断理解是否到位——**不比对标准释义** |
| b. 语境造句背 | `context_passage` | agent 用一批生词造一段短文（贴近用户关心的话题）→ 用户大致翻译 → 在语境中检验理解 |

**核心理念**：意思不该被存储，意思应该在复习时被**重新理解出来**。来源句是钥匙。

**调度**：由 FSRS 间隔重复算法决定"今日该复习哪些词"，避免生词本只进不出。

## 跨功能补充能力

1. **复习触发**：FSRS 调度器按遗忘曲线每日推送待复习词。
2. **错题毕业**：某类错误连续 N 次未再犯 → 标记 `resolved`，复盘不再唠叨。
3. **水平基线**：首次使用快速分级（CEFR / 估算雅思分），内容难度、造句、打分参照均基于此。
4. **生词反向应用**：功能2 写作/口语中检测"用上了刚学的词"，给正反馈。
5. **话题联动**：功能3b 造句贴近功能2 的话题，而非随机造句。

## 平台级功能（开源落地 + 习惯养成）

### MVP 内
- **首次配置向导**：选存储后端 → 配模型 provider/key → 测连通性 → 水平基线测试。新用户跑通的第一道关。
- **「今日学习」聚合首页**：把平行的三大功能串成日常 —— N 个待复习生词、上次练习待巩固的错题、推荐一个话题练习。一个 dashboard 告诉用户"今天该干什么"。
- **数据导入/导出**：整体 JSON 备份/迁移；生词本导出 **Anki**（CSV/.apkg），对接已有 Anki 工作流。

### 阶段 2
- **生词来源扩展**：除粘贴文本外，支持 URL 文章 / PDF / 字幕(.srt) 导入（看剧/读 paper 积累生词）。
- **难度自适应**：练习话题、造句难度随近期表现动态调整，而非固定在静态基线。
- **错误举一反三**：复盘时针对高频错误类型生成迷你专项练习（如时态老错 → 3 个时态填空），把错题本从"记录"变"训练"。
- **学习数据可视化**：背词曲线、错误类型分布、雅思分趋势。
- **生词/句子发音播放**：复用 TTS 适配器，背词时可听读音。
- **多语言界面 i18n**：UI 文案外置，便于国际贡献者翻译。
