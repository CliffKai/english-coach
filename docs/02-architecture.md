## 设计哲学：一切外部依赖皆适配器

存储、LLM、STT、TTS、发音评估 —— 全部通过**接口 + 适配器**抽象，业务代码只依赖接口。这让用户能自由切换云服务 / 自部署，也让第一版可以用最简单的适配器跑通，后续无痛升级。

## 五个 Agent 职责

| Agent | 对应功能 | 核心逻辑 |
|---|---|---|
| **TokenizerAgent** | 功能1 | 切词 → lemma → 去重 → 按水平过滤 → 逐词问询 → 不认识者连同来源句入库 |
| **TutorAgent** | 2a/2b | 引导式写作/口语，**练习模式=即时纠错+脚手架** |
| **ExaminerAgent** | 2c/2d | 自由写作/对话，**考试模式=延迟纠错**，结束后雅思/托福4维度打分 |
| **MemoryWordAgent** | 3a/3b | 3a 让用户说理解并判断；3b 用生词造短文供翻译检验 |
| **ErrorAnalysisAgent** | 2收尾 | 汇总本次+历史错误，模式识别，写复盘，回填错题本 |

## 延迟纠错的实现机制

考试模式下，ExaminerAgent 每轮回复时**内部**对用户输出做结构化错误标注（写入隐藏 buffer），但对用户只回自然对话/写作内容。用户点"完成/提前交卷"后，把 buffer 全量交给 ErrorAnalysisAgent 产出复盘。

## 模型按任务分配

不同任务用不同模型，兼顾能力与成本：

| 任务 | 推荐档位 | 理由 |
|---|---|---|
| 评分 scoring | 最强（如 Claude Opus） | 打分要准 |
| 引导/复盘 reasoning | 中档（如 Claude Sonnet） | 平衡 |
| 切词判断 tokenize | 轻量/本地（如 Ollama qwen） | 高频，省钱 |
| 高频对话 conversation | 中档/性价比模型 | 量大 |
